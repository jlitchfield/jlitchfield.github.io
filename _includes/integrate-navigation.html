The Business Problem

Taking into consideration the outcomes we’d like to achieve.

From three codebases to one

The Integrate product existed as three separate codebases within a single session.

With the unification of product experiences in Platform 3.0, we need a streamlined navigation system that aligns more effectively with user workflows, eliminates redundancies, and adapts to the upcoming permissions and product features.

This enhanced navigation will better serve our customers by simplifying access to tools, accommodating new access control models, and improving overall usability. Ultimately, the new system will cater to critical user roles, making users more efficient and enhancing their experience—helping to reduce churn.

Paint points and gaps

How might we remove friction and confusion?

Lack of ownership

Without a central authority, product managers often placed new functionality in their product area, resulting in scattered and inconsistent organization.

Buried critical functionality

Essential features were deeply hidden within the product, leading to decreased utilization.

Undiscoverable functionality

Features were buried deep within various navigation types, such as primary navigation → table → detail view → local navigation.

Incomprehensible visual design

Several features used tabs as a stepped process or a "poor man's wizard," causing users to encounter multiple errors if they didn't complete each tab correctly.

Non-standard jargon

Integrate used made-up jargon that wasn't industry standard, reducing comprehension and making content difficult to find.

Tabs as create flows

Users are shown a tabbed interface as create flows wherein the red tabs have something that needs to be completed.

Desk research

Design considerations that considers app footprint, user needs, and future state

Enhance the productivity and efficiency for all

A well-designed navigation will help both people complete their task quickly so that they can move onto something else. People should be able to quickly scan the navigation and understand which links are primary, secondary, and tertiary navigation items.

Make it consumable

Make it easy for users to understand, engage with, and derive value from. This involves several key elements that enhance usability, accessibility, and user satisfaction.

Clear Navigation: Users should be able to easily find their way through the product. Use common design patterns and logical flows.

Align navigation and copy with user goals

Avoid technical jargon, which risks diminishing conversion and overall product adoption rates, due to clunky usability. Well-thought-out copy will empower users to see the actual value of Integrate, improve engagement, and reduce churn rates.

Use unique visual design to establish hierarchy
  • Ensure that users can quickly scan the navigation to understand link hierarchy.
  • Use distinct visual designs to differentiate between primary, secondary, and tertiary navigation items.
  • Proper placement and grouping of links should clearly establish this hierarchy.
Consider footprint

While horizontal navigation is considered the most intuitive, consider vertical primary navigation for large, content-rich products. This allows for scrolling and efficient content access. Combine with horizontal navigation for limited secondary navigation needs and/or complex information architecture.

Add local navigation for related content

Prevent users from having to "pogo stick" up and down the hierarchy when frequently accessing related sections or completing multiple tasks within a single section.

Use location indicators

Engineers need to spend less time waiting for “the design phase” to end to start building. Designers need to create fewer static comps. Restricting these wasteful activities frees up more time to do many more valuable things.

Document as you design

Users should know where they are and where they came from. Help users orient themselves, especially when they are deep within the app.
